04 / ARTIFACT

A contribution is a structured, portable lesson pack.

A lesson pack, not a loose document.

A contribution is portable when identity, outcomes, teaching, assessment, evidence, access, rights, and provenance remain reviewable outside any particular agent or interface.

  1. IdentityStable lesson ID, version, risk tier, title, language, authors, outcome IDs.
  2. Learning contractPrerequisites, observable objectives, duration, misconceptions, mastery.
  3. Teaching sequenceExplanation, worked examples, practice, retrieval, transfer, recovery.
  4. EvidenceClaim-linked sources, confidence, route relevance, uncertainty, verification, use type, rights basis, and agent-access state.
  5. AssessmentChecks, answer logic, rubric, feedback, mastery and remediation criteria.
  6. ProvenanceAccountable principals, agent runs, instruction digests, original synthesis, assets, licences, and modification records.

Three contracts: the learning-content standard defines how explanations, examples, practice, feedback, and transfer work; Lesson Format v1 defines how that teaching is stored and rendered; the source and reuse policy defines original synthesis, compatible reuse, agent access, and rights review. The Format specimen is generated from the repository example and never counts toward coverage.

05 / QUORUM

Higher-risk lessons require more independent reviews.

More risk, more scrutiny.

MINOR CORRECTION2 + 1

1 academic and 1 accessibility/rights agent review, then 1 fresh adjudication run.

STANDARD LESSON3 + 1

1 academic, 1 learning-design, and 1 accessibility/rights agent review, then 1 fresh adjudication run.

HIGH IMPACT5 + 1

3 academic, 1 learning-design, and 1 accessibility/rights agent reviews, then 1 fresh adjudication run.

The number after “+” is not another review. It is a fourth, fresh agent run that receives the frozen candidate and complete review record, then commits the merge, revise, or reject rationale.

06 / STANDARD LESSON

A standard lesson passes three reviews, then one adjudication.

Three review agents. One adjudication agent.

  1. 01

    Academic

    Accuracy, scope, uncertainty, calculations, sources, and route relevance.

  2. 02

    Learning design

    Outcome, explanation, practice, assessment, mastery, and remediation align.

  3. 03

    Access + rights

    Accessibility, original synthesis, source and agent-access terms, attribution, asset licences, privacy, and provenance are verified.

  4. +1

    Adjudication

    A fresh fourth run records quorum, dissent, conditions, rationale, and decision.

07 / INDEPENDENCE

Review independence comes from isolated runs under one accountable operator.

Independent runs. One accountable operator.

Isolate each role

The standard quorum uses three distinct runs across three distinct providers: academic, learning design, and accessibility/rights. Review agents do not inherit the authoring conversation.

Disclose provenance

Every run records its operator, system, provider, model, version, run ID, and a SHA-256 digest of its material instructions. Provenance is disclosed and attested by the accountable operator, not cryptographically verified.

Freeze, then adjudicate

All reviews target one candidate commit. Lesson changes stale them. Adjudication uses a fourth run ID that appears nowhere in authoring or review provenance.
