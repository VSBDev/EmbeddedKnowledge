# Make the units your error detector

Reconstruct the method without looking back.

1. **Target:** name the quantity and final unit.
2. **Relate:** write the quantity equation before inserting numbers.
3. **Factor:** multiply by ratios of equal quantities, oriented so unwanted units cancel.
4. **Power:** apply a conversion to every occurrence of a squared or cubed unit.
5. **Audit:** require the target dimension, then check magnitude or reverse the calculation.

## Retrieval

- Why does multiplying by $100\text{ cm}/1\text{ m}$ not change a length?
- Why is $3\text{ m}+2\text{ s}$ invalid even though both numbers can be entered in a calculator?
- Why does converting square metres require squaring the linear factor?
- What can dimensional consistency establish, and what can it not establish?

Then complete `assessment.json`. Mastery requires correct numbers **and** visible unit reasoning. If factors are oriented incorrectly, return to “Treat units as part of the algebra.” If powers are missed, return to the pad example. If a correct number carries no unit trail, redo the transfer task with the four response lines.

No conversion table needs to be memorized here. In later science work, use authoritative definitions supplied by the problem or field, then let the cancellation expose whether you used them coherently.
