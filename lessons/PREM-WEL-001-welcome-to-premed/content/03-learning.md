# What learning will look like

Before you preview how the course will work, retrieve one reason for its order.

:::{check}
:id: check-sequence-why
:kind: retrieval

Pause here. Without looking back, cover the following text and commit a one-sentence answer before continuing: why does organic chemistry come before much of biochemistry, and why do inquiry and quantitative reasoning begin before both?
:::

Once you have answered, compare: organic chemistry prepares you to reason about carbon-based structures and reactions, while inquiry and quantitative reasoning provide methods for testing, measuring, and interpreting claims. If you repeated only the order, add why each step helps.

You will learn by meeting an explanation, retrieving it without looking, solving or modelling something, inspecting evidence, receiving feedback, and trying again with less support.

Practical work may be a **physical investigation**, **simulation**, or supplied **dataset or observation**. All can build question-design and analysis skills, but only physical work shows a hands-on action. Your record will name the kind completed; this welcome requires no physical work.

Here is a preview using a fictional training dataset. It requires no prior science. This previews the course's reasoning style; the welcome check does not score it.

:::{worked-example}
:id: worked-example-flow-preview
:label: A model-flow question from start to finish

**Task.** A simulation description says it uses the same liquid, pump setting, tube length, and ten-second collection time for two tubes. Tube A is narrow and Tube B is wider.

| Run | Tube A | Tube B |
|---|---:|---:|
| 1 | 20 mL | 79 mL |
| 2 | 19 mL | 82 mL |
| 3 | 21 mL | 78 mL |

The question is: *Does tube width affect the volume collected in this model?*

**Represent what is given.** Width is the planned difference. The description claims that liquid, tube length, pump setting, and collection time stay constant; the table reports volume. The pattern cannot verify those conditions.

**Choose a principle and plan.** A fair comparison changes the tested feature while keeping the other conditions the same. Compare all three runs, then keep the conclusion inside the simulation.

**Work through the evidence.** Tube A stays near 20 millilitres and Tube B near 80. Every Tube B result is about four times its paired Tube A result. The difference between tubes is much larger than the small run-to-run variation.

**Check the reasoning.** The units match and no run reverses the pattern. An investigator would inspect settings or logs to verify the claimed pump, liquid, tube length, and time; the data pattern cannot. If a log showed another difference, width would not be the only planned explanation.

**Interpret and limit.** The wider tube allowed more liquid to pass in this model under these settings. The data do not establish an exact law or tell us what happens in a human blood vessel. Later physics, chemistry, and physiology add pressure, resistance, fluids, tissues, and biological regulation.

**Self-explanation.** Pause here. Cover the following paragraph and commit an answer before continuing: which choice protects the conclusion most—collecting a large number, using millilitres, or holding the other conditions steady—and why?
:::

Once you have answered, compare: **holding the other conditions steady** protects the conclusion because that makes width the planned difference. Units make the measurement interpretable and repetition reveals variation, but neither fixes a comparison in which the pump or liquid also changed.

:::{check}
:id: check-faded-comparison
:kind: practice

Pause here. Cover the following paragraph and commit an answer before continuing. A second dataset compares a narrow tube at a low pump setting with a wide tube at a high setting. Can it isolate the effect of width? Answer yes or no, name the competing difference, and propose a repair.
:::

Once you have answered, compare: **No.** Pump setting changes with width, so either could explain the result. Repeat the comparison at one pump setting. If you named only the problem, add the repaired comparison.

This is the course's reasoning pattern in miniature: define the question, choose relevant ideas, work carefully, check, interpret, and state limits.
