# Redesign a peer study session

Four learners have forty minutes together. Each has classified three targets at baseline, but their fragile targets differ. They need to use the ideas independently in a seminar nine days later. Their old routine was to reread one shared chapter aloud.

The group setting changes the design problem. A shared explanation can be useful feedback, yet it can also conceal who generated which relation. Your plan must preserve individual evidence while using the group efficiently.

## Transfer task

Create a two-part plan: the forty-minute meeting and at least one later individual opportunity. Include all of the following:

1. how each learner selects a target from the baseline;
2. an individual retrieval operation before group feedback;
3. a criterion-based comparison that does not become answer copying;
4. targeted recovery for different error patterns;
5. a conditional use of mixed selection only where the seminar requires discrimination among already introduced ideas;
6. a later changed-context retrieval before day 9; and
7. a decision rule for a missing core relation, a correct but case-bound response, and a correct delayed response.

Do not solve by giving every learner the same topic or by asking the most confident learner to lecture for forty minutes. Those choices remove the baseline differences that make adaptation necessary.

## Model response after your attempt

One defensible plan uses four phases:

- **Minutes 0–4:** each learner selects one fragile target and writes the expected performance and criteria.
- **Minutes 4–12:** each learner completes a closed-source response individually. No peer answer is visible yet.
- **Minutes 12–28:** learners exchange responses and use the criteria to mark the first missing or unsupported relation. Each learner states a specific recovery action. Peers ask for the governing relation instead of supplying a polished answer to copy.
- **Minutes 28–40:** learners complete a short changed case or mixed-selection prompt only if the relevant categories are already introduced. Each learner then records an individual retrieval opportunity before day 9 in a different case or representation.

The decision rule is learner-specific. A missing core relation triggers a focused worked model and an earlier renewed retrieval. A correct response tied only to the original wording triggers a changed representation or case. A correct delayed response permits less support and a later cumulative check. None of the three observations fixes one universally correct next date.

## Why this is transfer

The worked example planned for one fictional learner over fourteen days. This task coordinates four different baselines, scarce shared time, peer feedback, and a nine-day horizon. The retrieval loop remains, but its allocation and safeguards must change. Merely replacing Ari's name with four names would not solve the coordination problem.
