# A design loop for durable performance

A retrieval-and-spacing plan is a chain of decisions, not a list of dates. Each opportunity must identify an operation, evidence criteria, a response to error, and a reason for its position in the sequence.

:::{definition}
:id: definition-target-performance
:label: Target performance

A **target performance** states what the learner will generate or do, without which supports, in what kind of context, and by what retention horizon. “Study membrane transport” names content; “compare two transport models without notes and justify the selection for a new case in twelve days” names a performance.
:::

:::{definition}
:id: definition-retrieval-practice
:label: Retrieval practice

**Retrieval practice** is an attempt to generate a relevant answer, explanation, prediction, representation, or procedure from memory before consulting the answer source. Copying a visible answer is production, but it is not retrieval because the source supplies the content during the attempt.
:::

:::{definition}
:id: definition-spacing
:label: Spacing and retention horizon

**Spacing** distributes learning or retrieval opportunities across time. A **retention horizon** is the whole future period over which the learner needs to retain and use the target knowledge. It constrains the plan. In the cited spacing literature, the technical **retention interval** is narrower: it is the time from the last practice opportunity to the later test. Neither term supplies one best gap for every learner and task.
:::

## The six-part loop

### 1. Specify the operation

Convert a topic label into observable performance. Name the conditions that matter. If the eventual task requires selection among models, recognition of model names is insufficient. If the eventual task requires an explanation, a list of isolated terms is insufficient.

### 2. Retrieve before review

Attempt the operation with the planned supports absent. The attempt may be short, but it must expose the reasoning you need to judge. A prompt such as “write every fact you remember” can miss the target if the goal is to select a model and justify it.

### 3. Compare with criteria

Use an answer model, rubric, or verified source after the attempt. Mark the first missing or incorrect relation, not only a total score. The location of the error makes the next action more specific.

### 4. Recover deliberately

Choose a recovery action that addresses the error: rebuild a missing prerequisite, compare two boundary cases, study one worked model, or correct an operation. Then retrieve again; rereading alone does not show whether the repair can be generated.

### 5. Retrieve after delay and variation

Place another opportunity after some time has passed, and change a feature that matters to the target performance. Variation might change the case, require a different representation, or remove a prompt. A changed number with identical reasoning is near practice; it does not by itself test adaptable use.

### 6. Apply a decision rule

A **decision rule** links an observation to the next step. For example: “If an essential relation is missing, use a focused worked model and retry a related case sooner. If the relation is reconstructed accurately after delay in a changed case, reduce support and place a cumulative check closer to the retention horizon.” This is an author-created planning heuristic, not a universal scheduling law.

## Block first when acquisition is incomplete; mix when selection is the goal

**Interleaving** mixes already introduced categories or strategies so the learner must choose among them. It can serve a discrimination goal when each category has a usable initial representation. If the learner is still acquiring the first procedure, a complete worked model and initially blocked practice may be the more coherent starting point. The design question is therefore “What operation must improve now?” rather than “Is mixed practice always better?”

## One representation, two reading routes

The table compresses the loop for auditing. The numbered sections above are its linear, nonvisual equivalent.

| Plan field | Question it answers | Evidence recorded |
| --- | --- | --- |
| Target performance | What must be generated, under which conditions, by when? | Explicit operation and criteria |
| Baseline | What happened on the most recent unaided attempt? | Error pattern, support, and timing |
| Retrieval task | What will the learner generate before review? | Learner response |
| Feedback and recovery | Which relation failed, why, and what happens next? | Criterion comparison and repair action |
| Spacing and variation | When and in what changed form will retrieval recur? | Later attempt under stated conditions |
| Decision rule | How will success or error alter the next opportunity? | A predeclared adaptation |
