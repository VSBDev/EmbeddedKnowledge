# Retrieve the map before reading the evidence

Complete this check without returning to the earlier map lesson. The aim is to expose what is available now, not to reward speed.

:::{check}
:id: check-map-prerequisite

1. In one sentence, distinguish a **knowledge node** from a **prerequisite link**.
2. Suppose node Gamma depends on node Beta, and node Beta depends on node Alpha. Gamma is currently inaccurate. Which node should you inspect first, and why?
3. Draw or type the route in dependency order using names and arrows or an ordered list.
:::

After committing a response, compare it with this routing criterion: a node is the bounded outcome being learned, whereas a prerequisite link says which earlier outcome supports access to it. Inspect the earliest dependency that may explain the current gap—Alpha in this route—rather than reopening every topic at once.

If you could state the distinction and reconstruct the dependency order, continue. If you could name the pieces but not use the direction, mark the prerequisite as fragile and revisit the dependency examples in `PREM-LPP-001`. If the terms were not available at all, return to that lesson's orientation and worked example before interpreting a baseline. None of these routes is a penalty.

Do not replace the attempt with a familiarity judgment such as “I remember seeing this.” The response itself is the evidence produced by this probe (`term-probe`).
