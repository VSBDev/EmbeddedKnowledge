# Retrieve the map before reading the evidence

Complete this check without returning to the earlier map lesson. The aim is to expose what is available now, not to reward speed.

:::{check}
:id: check-map-prerequisite

1. In one sentence, distinguish a **knowledge node** from a **prerequisite link**.
2. Suppose node Gamma depends on node Beta, and node Beta depends on node Alpha. Gamma is currently inaccurate, but the record contains no current evidence about Beta or Alpha. Which prerequisite should you probe first, and what result would justify probing Alpha next?
3. Draw or type the route in dependency order using names and arrows or an ordered list.
:::

After committing a response, compare it with this routing criterion: a node is the bounded outcome being learned, whereas a prerequisite link says which earlier outcome supports access to it. Because the stem supplies no evidence about either dependency, it does **not** justify declaring Alpha insecure. Probe Beta first: Beta is the nearest prerequisite and testing it adds the least unsupported recovery work. If Beta is insecure, probe Alpha. Reopen Alpha only if Alpha is then shown insecure; if Alpha is secure while Beta remains insecure, reopen Beta. If Beta is secure, keep Alpha closed and re-probe Gamma because the record does not implicate the earlier chain.

If you could state the distinction and use the evidence to decide which node to probe, continue. If you could name the pieces but not use the direction, record `topic-orientation-map-and-pathways` as a fragile prerequisite recovery target. If the terms were not available at all, record that graph outcome as unfamiliar. Neither route is a penalty, and neither depends on a lesson pack being published.

## In-scene recovery and fresh retry

Use this bounded restatement now: a **node** names one outcome. A **prerequisite link** points from an earlier supporting outcome to a later outcome. When the later outcome fails, work backward one link at a time, collect evidence, and reopen the earliest dependency that is actually shown insecure. Do not infer that every earlier node failed.

Now retry with a fresh chain: Zeta depends on Epsilon, and Epsilon depends on Delta. A current probe shows Delta secure. Epsilon produces a relevant but inconsistent response. Zeta is inaccurate. Which node should reopen first? **Epsilon** is the supported answer because it is the earliest dependency in this chain that the record actually shows insecure; Delta has current secure evidence. Cover this answer, explain the decision in your own words, and then re-attempt the Alpha–Beta–Gamma chain.

Do not replace the attempt with a familiarity judgment such as “I remember seeing this.” The response itself is the evidence produced by this probe (`term-probe`).
