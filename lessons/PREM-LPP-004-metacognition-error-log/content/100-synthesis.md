# Keep the log active, local, and revisable

Reconstruct the whole procedure as four verbs:

**monitor → diagnose → act → probe**

Monitoring preserves task conditions, the attempt before feedback, confidence, and rubric performance. Diagnosis locates the earliest unsupported operation while recording alternatives. Action targets that operation. A fresh probe tests the repair and updates the record.

## Decision summary

| Evidence pattern | Provisional interpretation | Proportionate next action | Fresh evidence |
|---|---|---|---|
| Requirement misread before an otherwise coherent response | Interpretation first unsupported | Restate conditions and contrast near-worded prompts | New prompt with altered wording |
| Relevant rule absent before feedback | Retrieval first unsupported | Brief explanation or restudy, then unaided reconstruction | Delayed recall plus application |
| Rules available but inapplicable model chosen | Selection first unsupported | Contrast model conditions and justify choice | New discriminating case |
| Applicable plan chosen but step applied incorrectly | Execution first unsupported | Complete and then independently perform the critical step | New values or representation |
| Result produced without a required plausibility check | Verification first unsupported | Practise the specific independent check | New case requiring the check |
| Intermediate evidence absent | Category unresolved | Expose steps in an accessible written or spoken-to-text probe | New record before targeted repair |

Read linearly, each row states evidence, hypothesis, action, and fresh probe. The table is a route guide, not an exhaustive taxonomy.

## Confidence route

For each matched task, compute predicted rubric percentage minus observed rubric percentage. Describe the direction and size on that task. Look for a pattern across comparable, delayed, or varied probes before changing the confidence routine. If predictions repeatedly exceed performance, require the learner to cite observable response evidence before predicting a score. If predictions repeatedly fall below strong performance, compare the completed response with the rubric and schedule another independent verification. When confidence and performance do not use the same scoring rule, keep them separate. Research using repeated trial-level quadratic and signal-detection measures shows that confidence–accuracy correspondence can respond to targeted feedback, but those measures are not this lesson's $G$; no one study or log establishes how the signed gap will change or guarantees general transfer.

## Recovery route

- If you cannot reconstruct the prior durable-performance loop, return to `topic-orientation-mastery-and-retrieval` and produce target → retrieve → compare → repair → retrieve later.
- If you can compute a gap but cannot classify the error, revisit the five-operation chain and practise locating the earliest supported and unsupported links.
- If you classify confidently from a final score alone, use the unresolved Cedar case and design a probe that exposes intermediate steps.
- If your actions remain broad, force a one-sentence link: “Because the record first fails at ___, the next activity will exercise ___, and the fresh probe will show ___.”
- If a response format caused the apparent failure, change the medium while preserving the reasoning. A barrier in handwriting, visual layout, speech, pointing, or speed is not automatically a content error.

## Delayed link to the study plan

Carry forward only the current action, fresh-probe timing, and relevant task ID into `topic-orientation-study-plan`. The Premed graph makes this lesson a prerequisite for that planning outcome. Reopen the evidence after the fresh probe and revise the plan; do not freeze the original category into a learner profile.
