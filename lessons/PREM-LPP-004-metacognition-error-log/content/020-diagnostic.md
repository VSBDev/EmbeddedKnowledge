# Retrieve the durable-performance loop

Attempt this low-stakes check before reopening the prerequisite lesson. The purpose is to reveal which part of the prior model is available now.

:::{check}
:id: check-prerequisite-loop

Without notes, write an ordered study loop that includes:

1. the performance to be produced;
2. what happens before feedback;
3. how an error changes the next action; and
4. what later evidence is needed before a durable-performance judgment.

Then answer: where would a confidence judgment enter the loop, and why can it not replace the performance record?
:::

Commit your answer before reading the criterion below.

## Criterion and route

A usable reconstruction names a target performance, an unaided retrieval attempt, comparison with an answer model or rubric, a repair selected from the error, and another retrieval after a delay or bounded variation. Confidence can be recorded beside the attempt, but it becomes informative only when compared with the observed result.

If your sequence stops at rereading, return to `PREM-LPP-003` and reconstruct target → retrieve → compare → repair → retrieve later. If you included an attempt and feedback but not a later probe, revisit its spacing and durable-performance sections. If the loop is available but confidence feels separate, continue: this lesson will connect the two records explicitly.

The graph places `topic-orientation-metacognition-and-error-log` after `topic-orientation-mastery-and-retrieval` and before `topic-orientation-study-plan`. This means the error record consumes performance evidence from the prior outcome and supplies decisions to the later plan; it does not mean every learner must use one fixed schedule.
