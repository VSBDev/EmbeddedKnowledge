# A connection is not always an order

## Commit to a prediction

Suppose a map contains `A → B` as a prerequisite and a cross-link between B and C. Does the map require the order A, then B, then C? Answer before reading on.

:::{misconception}
:id: misconception-every-edge-is-order

An understandable but inadequate rule is: “Every visible connection tells me what comes next.” It works for the A-to-B prerequisite, then fails at the B-to-C cross-link. The cross-link records conceptual relevance without imposing sequence, so it cannot place C after B.

A more adequate rule first reads the **edge type**. Use prerequisite edges for ordering, containment for hierarchy, and cross-links for integration or later comparison. On the stated evidence, only A before B is required; C's eligibility depends on C's own prerequisites.
:::

## Recheck in a changed case

Topic D and Topic E share a module. E has no declared prerequisite, while D has a cross-link to E. Which topic must come first?

The map fragment imposes no order between them. Choose by the learner's goal, route, and other prerequisite records. Sharing a container or a cross-link is not a hidden dependency.

The same discipline applies to labels. A high placement-confidence label does not mean “study immediately,” and a route tag does not mean “prerequisites satisfied.”
