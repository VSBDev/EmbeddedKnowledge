# Your map-reading card

Before choosing a next topic, write five lines:

1. **Identity:** target stable topic ID and observable outcome.
2. **Scope:** current route-instrumentation status, then requirement and effective route membership.
3. **Order:** unmet prerequisite chain, read from prerequisite to dependent.
4. **Evidence:** what the placement labels support and what they do not.
5. **Action:** the earliest relevant outcome whose prerequisites are secure, plus the evidence used to judge “secure.”

Use cross-links after or alongside this plan to connect ideas; do not silently turn them into locks. Treat a non-empty route-tag set as a content filter until the current curriculum version confirms that the documented route is instrumented. Keep institution-specific requirements in a dated external audit rather than inferring them from graph tags.

## Accessibility and alternatives

Every relationship in this lesson is written as a sentence or table row; no answer depends on color, coordinates, visual tracing, audio, hover, dragging, or time pressure. A learner may respond with an ordered list, a table, or spoken text transcribed by an accessible tool. The operation remains the same: identify the node, name the link type, justify scope, and select a next action.

## Recovery route

- If hierarchy is unstable, rebuild three lines—domain, module, topic—and retry `item-map-order` with new labels.
- If order is unstable, translate each edge into a sentence before retrying `item-cross-link-order` with a different fragment.
- If scope labels are overinterpreted, write “placement, not truth or admission” beside `evidenceConfidence`, then retry `item-label-boundary` in your own words.
- If the full plan fails, return to the worked example, mark the first unsupported inference, and retry `item-route-plan` with one fewer hint. Unlimited untimed attempts are permitted.
