# Turn the plan into a weekly control loop

The four passes build one week, but the outcome requires more than a static calendar. The plan becomes sustainable in the bounded sense when it forms a control loop:

1. **Plan:** bound capacity, place dependencies, distribute functions, and protect reserve.
2. **Perform:** attempt the scheduled learning operations and record the evidence actually produced.
3. **Inspect:** compare completion, correctness, explanation, transfer, confidence, and access with what the plan expected.
4. **Revise:** protect, move, split, replace, or remove work from the next plan.

## Weekly review from memory

Before opening the old plan, reconstruct:

- the capacity boundary;
- the five required functions;
- the four passes;
- the strongest current error-to-action chain; and
- the rule that prevents automatic backlog.

Then inspect the record. Ask:

| Question | Decision it supports |
| --- | --- |
| Which required or prerequisite work still lacks evidence? | Protect or move |
| Which task was too large but still coherent when divided? | Split |
| Which medium failed while the learning operation remained required? | Replace with an equivalent route |
| Which work became redundant, optional, or unrelated to the mapped outcome? | Remove |
| Which earlier learning now needs a changed retrieval prompt? | Schedule delayed review |
| Which constraint needs explicit reserve or a reduced route? | Protect recovery capacity |

The next plan is not a moral score for the previous week. It is a revised model based on evidence and constraints.

## Delayed retrieval link

At the start of the next science module, build its first week without opening this template. Explain why each problem, lab, and retrieval contact occupies its position. Later lessons should reactivate this outcome whenever an error log changes the order or amount of prerequisite work.

If disruption leaves only the course's minimum viable route (`term-minimum-viable-study-day`), record that reduced evidence truthfully and begin the next review from it. Do not mark the abandoned full plan as complete, and do not dump its entire task list into the next week.
