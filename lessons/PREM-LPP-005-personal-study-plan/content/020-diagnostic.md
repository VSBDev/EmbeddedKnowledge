# Retrieve the evidence-to-action chain

Do this check before reading the feedback. It is low stakes and does not ask for private schedule details.

:::{check}
:id: check-error-to-action

Choose one fictional or real-but-anonymized weak curriculum node. From memory, write:

1. the node or a generic label for it;
2. the error class shown by the latest attempt;
3. the earliest insecure prerequisite, if one is implicated;
4. one targeted next action; and
5. the fresh evidence that would show whether the action worked.
:::

## Check after attempting

A complete chain might read:

> Node: interpreting a line graph. Error: `REP`, because the verbal trend was accurate but the axes were reversed. Earliest insecure prerequisite: identifying what each axis measures. Next action: complete one contrasting graph example and explain each axis before interpreting the trend. Fresh evidence: interpret a different graph without prompts, then retrieve the axis rule later.

The important relation is not the particular error code. The next action follows from the observed error and its prerequisite. “Study harder” is too broad because it does not specify what representation, relation, or decision will change.

If you can produce that chain, carry its targeted action into the weekly plan. If you cannot, do not invent a priority from confidence or urgency alone.

## Recovery route

Return to the prerequisite outcome, Metacognition and error logs (PREM-01.04). Classify one missed response, identify its earliest insecure prerequisite, and choose one contrasting repair task plus fresh evidence. Then repeat this diagnostic. Missing a prerequisite is routing evidence, not failure.

:::{source-note}
:claims: claim-error-directed-revision
:sources: source-premed-syllabus

The course recovery protocol supports the node-to-error-to-prerequisite-to-repair chain used by this diagnostic. It does not establish the personal cause of a missed response.
:::
