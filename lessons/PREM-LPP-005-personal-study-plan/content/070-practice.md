# Fade the planning support

Attempt each task before reading its feedback. The support decreases from a nearly complete ledger to an independent plan.

## Task 1: complete the missing decisions

A fictional learner has eight usable blocks. A fixed supervised lab uses two. The draft already contains one concept block, one preparation block before the lab, the lab, and one debrief block after it. One cumulative retrieval contact is due. The error log names no current prerequisite gap.

Complete these decisions:

1. Where should the due retrieval be placed, and what must its entry say?
2. How many blocks remain after scheduling it?
3. How will recovery capacity be represented?

### Feedback

The retrieval must occur after its earlier learning episode, ask for an attempt before feedback, and name the earlier target and evidence. The stated draft uses five blocks: concept, preparation, two lab blocks, and debrief. One retrieval block makes six, leaving two blocks. Protecting both as reserve is defensible; scheduling one problem block and leaving one reserve is also defensible if the problem serves the current outcome and the eight-block boundary is not exceeded. The key is to distinguish scheduled study from genuine reserve.

## Task 2: choose the stronger sequence

The error log now shows a repeated graph-translation error. Compare:

- **Sequence X:** mixed problem set → graph-translation explanation → retry;
- **Sequence Y:** contrasting graph representations → explanation of the governing relation → mixed problem → later retrieval.

Choose the stronger initial route and explain when you would fade its support.

### Feedback

Sequence Y is stronger because it repairs the earliest evidenced gap before dependent mixed selection. Fade the contrasting-example scaffold after the learner independently selects and explains the graph relation in varied problems. Keep a later retrieval contact because immediate success is not the same as delayed retention.

## Task 3: construct independently

Use this fictional evidence record:

- eleven usable blocks;
- one two-block fixed supervised practical;
- one concept prerequisite and one preparation task required before it;
- one debrief after it;
- a `MOD` error in selecting a model for a dependent problem;
- one delayed retrieval contact due;
- one optional enrichment task; and
- one block that may lose connectivity.

Construct a chronological text ledger. For every scheduled activity, name its learning function, dependency or timing reason, evidence, and contingency. Protect recovery capacity. At weekly review, give one protect, move, split, replace, or remove decision for the optional enrichment.

### Self-check and next step

A defensible response:

- uses no more than eleven blocks;
- places concept and preparation before the supervised practical and debrief after it;
- repairs the model-selection error before another dependent problem;
- includes a later retrieval attempt before feedback;
- represents concept, problem, lab, review, and recovery functions;
- prepares an offline text route for the connectivity risk; and
- removes or defers optional enrichment if required or error-linked work needs the capacity.

If one relation is missing, repair only that relation and retry with a new order. If several are missing, return to the four-pass ledger and annotate one pass at a time. If the task is already easy, hide the checklist and solve the transfer scene without it.
